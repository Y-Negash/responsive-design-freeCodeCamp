Project Overview

The Trading Journal application is designed to help users manage and track their trades and associated issues. This system enables efficient record-keeping of trading activities, supporting detailed trade data and optional issue descriptions for each trade. With a structured approach to tracking, it provides users with valuable insights for refining trading strategies over time.

Tech Stack

Frontend: Typescript and React

Backend: Java with Spring Boot

Database: PostgreSQL

Installation

Prerequisites:

Java for backend.

Node.js and npm for frontend.

PostgreSQL for database management.

Setting it up:

  • Clone the repository:
  • git clone https://github.com/Y-Negash/Trading_Journal.git
    cd Trading_Journal
  • Backend Setup:
  • Navigate to the backend folder and install dependencies.
  • Ensure the database URL, username, and password are correctly set in the application properties file.
  • Frontend Setup:
  • Navigate to the frontend folder and install dependencies using npm install
  • Start development server using npm start
    • API Documentation

    Trade Endpoints

  • /trades - Get all trades (GET method)
  • /trades/{id} - Get trades by id (GET method)
  • /trades - Create trades (POST method)

    • Issue Endpoints

  • /issues/date/{tradeDate}Get issues by date (GET method)
  • /issues - Add issue to trade (POST method)
